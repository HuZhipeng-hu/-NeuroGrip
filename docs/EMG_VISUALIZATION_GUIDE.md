## 🎨 新增功能详解

### 1. EMG 8通道折线图

**位置**: 实时监控页面第二部分

**功能**:
- 同时显示8条EMG信号曲线，每条曲线颜色不同
- X轴：时间轴（最近200个数据点）
- Y轴：信号幅度（0-255）
- 显示网格背景便于读数
- 图例显示各通道颜色对应

**技术实现**:
```typescript
// 数据缓存
@State emgHistory: number[][] = [[], [], [], [], [], [], [], []]

// 绘制方法
private drawEmgChart(ctx: CanvasRenderingContext2D, width: number, height: number)

// UI中的Canvas组件
Canvas()
  .width('100%')
  .height(240)
  .onReady((event: CanvasRenderingContext2D) => {
    this.drawEmgChart(event, 360, 240);
  })
  .key(`emg-chart-${this.refreshCanvas}`)  // 刷新标志
```

**颜色编码**:
- CH1: 蓝色 (#3B82F6)
- CH2: 绿色 (#10B981)
- CH3: 琥珀色 (#F59E0B)
- CH4: 红色 (#EF4444)
- CH5: 紫色 (#8B5CF6)
- CH6: 粉红色 (#EC4899)
- CH7: 青色 (#14B8A6)
- CH8: 靛蓝 (#6366F1)

**显示信息**:
- 折线图下方显示当前各通道实时数值
- 实时更新，无延迟

### 2. 3D 手势位置可视化

**位置**: 实时监控页面第三部分

**功能**:
- 利用IMU传感器数据（陀螺仪、加速度、角度）绘制3D手臂姿态
- 显示一个旋转的3D立方体，代表手臂的空间位置
- 等角投影（Isometric Projection）显示3D效果
- 实时显示Pitch、Roll、Yaw三个旋转角度

**技术实现**:

```typescript
// 3D旋转公式（欧拉角）
// 1. 绕X轴旋转（Pitch）
y1 = y * cos(pitch) - z * sin(pitch)
z1 = y * sin(pitch) + z * cos(pitch)

// 2. 绕Y轴旋转（Roll）
x2 = x * cos(roll) + z1 * sin(roll)
z2 = -x * sin(roll) + z1 * cos(roll)

// 3. 绕Z轴旋转（Yaw）
x3 = x2 * cos(yaw) - y1 * sin(yaw)
y3 = x2 * sin(yaw) + y1 * cos(yaw)

// 等角投影变换
screenX = centerX + (x3 - z2) * scale * 0.866
screenY = centerY + (y3 - (x3 + z2) * 0.5) * scale
```

**显示信息**:
- 顶部显示加速度信息 (X, Y, Z)
- 中间大型3D立方体展示
- 下方显示陀螺仪信息 (X, Y, Z 角度)
- 底部实时显示 Pitch、Roll、Yaw 角度值

**坐标系说明**:
- **Pitch (俯仰角)**: 绕X轴旋转，范围 -90° 到 90°
  - 正值：手臂向上抬起
  - 负值：手臂向下放低
  
- **Roll (横滚角)**: 绕Y轴旋转，范围 -180° 到 180°
  - 正值：手臂向右倾斜
  - 负值：手臂向左倾斜
  
- **Yaw (偏航角)**: 绕Z轴旋转，范围 -180° 到 180°
  - 正值：手臂逆时针旋转（从上往下看）
  - 负值：手臂顺时针旋转

## 📈 数据流和性能优化

### 数据收集机制

```
WebSocket接收EMG数据
   ↓
更新emgChannels（当前值）
   ↓
将当前值推送到8个emgHistory数组
   ↓
如果emgHistory长度>200，移除最早的点
   ↓
更新angleHistory（同步更新）
   ↓
触发refreshCanvas刷新标志
   ↓
Canvas自动重绘（key binding）
```

### 性能考虑

1. **数据缓存大小**: 200个点
   - 平均接收频率：50Hz
   - 缓存时间窗口：4秒
   - 内存占用：约 12KB

2. **Canvas刷新**:
   - 使用`key`绑定 `refreshCanvas` 状态
   - 每次收到新数据时翻转该状态
   - 自动触发Canvas重绘，但不会过度渲染

3. **计算复杂度**:
   - EMG图表: O(n) 其中n=200，绘制8条线
   - 3D绘制: O(1) 常数时间（立方体12条边）

## 🐛 故障排查

### 问题1：Canvas没有渲染

**症状**: 图表区域为空白或只显示背景色

**解决**:
1. 检查WebSocket连接是否成功
2. 验证数据是否到达（查看控制台日志）
3. 确保`onReady`回调被执行
4. 检查`refreshCanvas`状态是否在更新

### 问题2：3D立方体显示错误

**症状**: 立方体显示不正确或扭曲

**解决**:
1. 检查IMU数据范围（角度应为 -180° 到 180°）
2. 验证欧拉角旋转顺序正确
3. 尝试调整`scale`参数（改为15-20）

### 问题3：折线图波动过大

**症状**: 曲线抖动或信号不清晰

**解决**:
1. 增加`MAX_HISTORY_POINTS`（改为300-500）
2. 检查EMG信号质量
3. 检查是否有电磁干扰
4. 验证采样率是否正确
